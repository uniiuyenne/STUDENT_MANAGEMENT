# Screens
This folder contains all application screens/pages.

## Structure
- `add_edit_student/` - Student creation and editing screen
- `student_list/` - Main student list view
- `student_detail/` - Detailed student information
- `statistics/` - Analytics and statistics dashboard

## Guidelines
- Each screen should be a StatelessWidget or StatefulWidget
- Use providers for state management
- Keep UI logic separate from business logic
